# 需求文档

## 1. 建设目标

面向高校人才培养与就业服务需求，建设基于 Spark 的就业市场需求分析与学生就业能力辅助平台。平台以公开招聘岗位为真实主数据，以学生主动填写或高校授权导入的匿名数据为补充，形成岗位需求分析、学生岗位推荐和专业培养参考三条业务主线。

## 2. 数据边界

当前无法取得高校真实毕业生去向数据，因此项目遵循以下边界：

- 不伪造真实毕业生就业率、签约率、薪资或培养质量结论。
- 学生侧数据由学生自主填写，用于个人推荐和能力差距分析。
- 高校侧支持匿名模板导入；没有授权数据时只展示明确标记的演示队列。
- 不建设就业跟踪、签约率和毕业去向调查模块。
- 招聘会仅使用公开场次与规模统计，不关联学生参会或签约结果。
- 当前岗位时间跨度较短，只输出需求现状、近期信号和场景推演，不宣称长期人才需求预测。

## 3. 转换后的核心功能

### 3.1 就业市场数据管理

- 多来源公开岗位采集、质量检查、去重和标准化。
- HDFS 分层存储、Spark 批处理、MySQL 业务查询。
- 城市、行业、岗位方向、学历、经验、薪资和技能需求分析。

### 3.2 学生就业服务

- 学生画像、技能等级和就业期望维护。
- Top10 岗位推荐、各维度匹配分和推荐理由。
- 技能差距、提升优先级和目标岗位查询。

### 3.3 高校培养参考

- 匿名学生队列或演示队列的能力供给统计。
- 专业技能供给与市场技能需求对齐度。
- 目标岗位覆盖率、常见技能缺口和课程训练方向建议。
- 输出仅作为专业建设、招生规划和就业指导的辅助证据，不直接评价培养质量。

### 3.4 可行的增强模块

- 区域产业人才需求矩阵：按地区、行业、岗位方向和技能交叉分析需求规模。
- 专业—岗位—技能对齐：结合专业岗位映射字典，分析目标岗位覆盖与技能需求。
- 培养方向场景推演：调整目标岗位或技能组合后，比较岗位覆盖量和匹配变化。
- 招聘会公开数据：分析场次、地区、企业数、岗位数和计划招聘人数。
- 数据质量与来源看板：展示批次、来源、缺失率、清洗量和更新时间。
- 历史岗位批次：积累足够时间跨度后，再启用趋势建模和模型回测。

## 4. 验收口径

- 真实岗位链路可以从采集、Spark 处理运行到 API 和前端展示。
- 学生可完成画像维护、岗位推荐和技能差距查看闭环。
- 高校端所有统计均能标明真实、匿名导入或演示数据来源。
- 所有预测或建议均展示数据时间范围、样本量和适用边界。
